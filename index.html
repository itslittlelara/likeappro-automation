Configuração
Como montar no n8n
Webhook Recomendado
App envia POST para o n8n a cada mudança de status. Reação em tempo real.
Polling
n8n consulta o banco de dados a cada X minutos. Útil sem acesso ao dev do app.
1
Criar o nó Webhook no n8n
Novo workflow → nó Webhook → copiar URL gerada e passar para o dev do app
2
Configurar o nó Switch por evento
Filtrar pelo campo "event" e rotear para o canal certo: email ou WhatsApp
3
Conectar os nós de envio e testar
Cada branch aponta para Email ou HTTP Request da API escolhida — testar com um cadastro real

Roadmap
Próximos passos
01
Mapear status no banco
Confirmar com o dev quais eventos já existem e quais precisam ser criados ou adaptados
02
Escolher a API de WhatsApp
ManyChat para facilidade, Z-API para agilidade, Evolution se tiver dev disponível
03
Montar o primeiro fluxo
user.created → WhatsApp. Só um fluxo, para validar que a mensagem chega corretamente
04
Expandir os triggers
Um por vez: inatividade → cancelamento → reativação → churn
05
Monitorar e otimizar
Após 2 semanas de dados, analisar taxas de abertura e resposta para refinar as mensagens e os timings
